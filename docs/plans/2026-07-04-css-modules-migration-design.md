## 1. Архитектурный контекст и проблематика

### Проблема инлайн-стилей (Inline Styles)
В текущей реализации компоненты страниц (`Dashboard.tsx`, `DeckEditor.tsx`, `StudySession.tsx`) содержат сотни строк инлайновых стилей вида `style={{ display: 'flex', gap: '20px', ... }}`.
Это приводит к следующим архитектурным проблемам:
1. **Нарушение принципа разделения ответственности (Separation of Concerns):** Разметка, бизнес-логика и стили смешаны в одном файле. Это усложняет чтение кода и отладку.
2. **Сложность повторного использования:** Стили нельзя переиспользовать между элементами без дублирования кода.
3. **Отсутствие поддержки медиа-запросов и псевдоклассов:** В инлайн-стилях невозможно напрямую использовать `:hover`, `:focus` или `@media` без привлечения JS-логики состояния.

---

## 2. Предлагаемое архитектурное решение: CSS Modules

**CSS Modules** — это подход, при котором все классы в CSS-файле по умолчанию имеют локальную область видимости (scoped). Сборщик (в нашем случае Vite) автоматически трансформирует имена классов, делая их уникальными во всем приложении (например, `.container` превращается в `.Dashboard_container__a8f3g`).

### Преимущества для архитектуры:
* **Инкапсуляция стилей (Style Encapsulation):** Полностью исключаются конфликты имен. Разработчик может назвать класс `.wrapper` в десяти разных компонентах, не боясь, что стили пересекутся.
* **Интеграция с дизайн-системой:** Модули CSS имеют беспрепятственный доступ к глобальным CSS-переменным, объявленным в `:root` и `html[data-theme]`.
* **Чистый JSX:** Код компонентов становится декларативным и легко читаемым.

## 3. Архитектура интеграции

### 3.1 Глобальный уровень (Global Scope)
Файлы `variables.css`, `index.css` и `components.css` остаются глобальными. Они содержат:
- Дизайн-токены (цвета, шрифты, скругления, тени).
- Общие сбросы стилей (CSS Reset) и базовую разметку `body`/`html`.
- Глобальные атомарные компоненты (например, `.btn`, `.flashcard`).

### 3.2 Локальный уровень (Local/Component Scope)
Каждая страница получает свой файл `.module.css`:
- `Dashboard.tsx` -> `Dashboard.module.css`
- `DeckEditor.tsx` -> `DeckEditor.module.css`
- `StudySession.tsx` -> `StudySession.module.css`

Все классы внутри этих модулей описывают структуру и сетку конкретных страниц.
