# Разработка MindFlip — приложения с интервальными повторениями (SRS)

В рамках этого плана мы создадим современное веб-приложение MindFlip для обучения по флеш-карточкам. Приложение будет адаптировано для изучения программирования (JS, Angular) и поддерживать отображение блоков кода с подсветкой синтаксиса.

## Принятые архитектурные решения (по итогам /grill-me)
1. **Технологический стек:** React + Vite (быстрая сборка, современная экосистема).
2. **Стилизация:** Vanilla CSS (полный контроль над микро-анимациями и кастомными компонентами).
3. **Хранение данных и Авторизация:** Firebase (Firestore для карточек и статистики, Authentication для входа), тариф Spark (бесплатный).
4. **Алгоритм (SRS):** Упрощенная система Лейтнера (Leitner System). Карточки распределяются по уровням (уровень 1 — повторение каждый день, уровень 2 — раз в 3 дня и т.д.). У пользователя только 2 кнопки: «Помню» (повышает уровень) и «Забыл» (сбрасывает на 1 уровень).
5. **Дизайн:** Геймифицированный (яркие сочные цвета). Маскот: Хамелеон Леон.
6. **Отображение контента:** Карточки будут поддерживать Markdown и подсветку синтаксиса (например, с использованием `react-markdown`), чтобы код на JS и Angular выглядел красиво и читабельно.
7. **Адаптивность (Responsive Design):** Mobile-First подход. Интерфейс должен идеально выглядеть на смартфонах и плавно масштабироваться (Grid/Flexbox + Media Queries) для удобного использования на больших десктопных мониторах.

## Структура проекта (Proposed Architecture)

### 1. Ядро алгоритма (Algorithm)
- **`src/lib/leitner.ts`**
  Реализация алгоритма Лейтнера. Функция принимает карточку и ответ пользователя, рассчитывая следующий уровень и дату следующего повторения.

### 2. Интеграция с Firebase (Backend Services)
- **`src/lib/firebase.ts`**
  Инициализация Firebase приложения.
- **`src/services/auth.ts`**
  Методы для регистрации и авторизации.
- **`src/services/db.ts`**
  Работа с базой данных Firestore (колоды, карточки, статистика).

### 3. Компоненты UI (Gamified Design System)
- **`src/styles/index.css`** & **`src/styles/variables.css`**
  Глобальные стили, CSS-переменные для цветовой палитры.
- **`src/components/ui/Button.tsx`**
  Анимированные тактильные кнопки.
- **`src/components/cards/Flashcard.tsx`**
  Интерактивная карточка с поддержкой Markdown и подсветкой фрагментов кода.
- **`src/components/mascot/Leon.tsx`**
  Компонент Хамелеона Леона (изменение цвета в зависимости от ответа).

### 4. Основные страницы (Pages)
- **`src/pages/Dashboard.tsx`**
  Главный экран со списком колод и статистикой.
- **`src/pages/StudySession.tsx`**
  Экран изучения. Вместо 4 кнопок SM-2 будут 2 крупные кнопки ("Помню" и "Забыл").
- **`src/pages/DeckEditor.tsx`**
  Интерфейс для добавления карточек с предпросмотром Markdown-кода.

## План выполнения и Стратегия Git (Execution Steps)
Разработка будет вестись строго итеративно. После каждого логического шага будет создаваться отдельный Git-коммит:

1. **Инициализация** (Commit: `chore: init react vite project with dependencies`)
   - Запуск `npm create vite@latest`, инициализация репозитория `git init`.
   - Установка зависимостей: `firebase`, `react-router-dom`, `react-markdown`.
2. **Базовый UI-кит** (Commit: `feat: setup gamified UI and css variables`)
   - Настройка глобальных стилей и создание анимированных тактильных кнопок.
3. **Компонент Карточки** (Commit: `feat: add Flashcard with Markdown support`)
   - Реализация карточки, способной отображать блоки кода с подсветкой синтаксиса.
4. **Алгоритм Лейтнера** (Commit: `feat: implement Leitner system logic`)
   - Написание функции расчета интервалов в `src/lib/leitner.ts`.
5. **Основные Экраны** (Commit: `feat: add main pages and routing`)
   - Сборка экранов Dashboard, StudySession и DeckEditor.
6. **Интеграция Firebase** (Commit: `feat: integrate Firebase data persistence`)
   - Подключение Firestore для сохранения прогресса в облаке.
